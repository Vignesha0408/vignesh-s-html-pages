Cryptography - Quick Reference

ConceptDescription
Definition
Practice and study of techniques for secure communication in presence of adversaries. Protects data confidentiality, integrity, and authenticity.
Goals
Confidentiality, Integrity, Authentication, Non-repudiation, Key exchange
Terminology
Plaintext (original), Ciphertext (encrypted), Encryption (encoding), Decryption (decoding)
Symmetric CryptographyDescription
Definition
Uses same key for encryption and decryption. Fast but key distribution is challenging.
Algorithms
AES (Advanced Encryption Standard), DES, 3DES, Blowfish, RC4, ChaCha20
Modes
ECB, CBC, CFB, OFB, CTR (Counter mode)
Use Cases
Disk encryption, bulk data encryption, secure file transfer
Asymmetric CryptographyDescription
Definition
Uses public-private key pairs. Public key is shared, private key is secret. Solves key distribution problem.
Algorithms
RSA, ECC (Elliptic Curve), DSA, ElGamal, Diffie-Hellman
Use Cases
Digital signatures, key exchange, secure email, SSL/TLS
Hash FunctionsDescription
Definition
One-way functions that convert input of any size to fixed-size output. Deterministic and irreversible.
Properties
Pre-image resistance, Second pre-image resistance, Collision resistance
Algorithms
SHA-256, SHA-3, MD5 (deprecated), SHA-1 (deprecated), BLAKE2
Use Cases
Password storage, Data integrity verification, Digital signatures, Blockchain
Digital SignaturesDescription
Definition
Mathematical scheme for verifying authenticity and integrity of digital messages or documents.
Process
Hash message, encrypt hash with private key, recipient decrypts with public key and compares hash
Standards
RSA-PSS, DSA, ECDSA, EdDSA
Key ExchangeDescription
Diffie-Hellman
Method for securely exchanging cryptographic keys over public channel. Based on discrete logarithm problem.
ECDH
Elliptic Curve Diffie-Hellman. More efficient version using elliptic curves.
RSA Key Exchange
Server sends public key, client encrypts session key with it.
ApplicationsDescription
SSL/TLS
Hybrid cryptosystem using both symmetric and asymmetric encryption for secure web browsing.
PGP/GPG
Uses public-key cryptography for email encryption and digital signatures.
Blockchain
Uses hashing, digital signatures, and public-key cryptography for security.
VPN
Virtual Private Networks use encryption for secure remote access.
Security ConsiderationsDescription
Key Length
AES-128/192/256, RSA-2048/4096 bits. Longer keys provide more security but slower performance.
Quantum Threat
Quantum computers could break current asymmetric algorithms. Post-quantum cryptography is being developed.
Implementation
Side-channel attacks, timing attacks, fault injection can compromise secure algorithms.
