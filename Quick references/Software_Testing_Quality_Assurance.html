Software Testing & Quality Assurance Cheat Sheet

Testing Levels

LevelDescriptionFocusPerformed ByTools
Unit TestingTesting individual components or modulesFunctionality of specific code unitsDevelopersJUnit, NUnit, PyTest, Mockito
Integration TestingTesting interaction between modulesModule interfaces and data flowDevelopers/TestersTestNG, Selenium, Postman
System TestingTesting complete system functionalityEnd-to-end system behaviorTestersSelenium, QTP, LoadRunner
Acceptance TestingTesting system against business requirementsBusiness requirements fulfillmentEnd users/StakeholdersCucumber, FitNesse, UAT tools

Testing Types

TypeDescriptionWhen to UseExample
Functional TestingTesting specific functionality of the applicationDuring development and maintenanceTesting login functionality
Non-Functional TestingTesting performance, usability, reliabilityAfter functional testingLoad testing, stress testing
Regression TestingEnsuring new changes don't break existing functionalityAfter code changes or new featuresRe-running existing test cases
Smoke TestingBasic tests to verify critical functionalityAfter each build or deploymentBasic login, navigation tests
Sanity TestingTesting specific functionality after minor changesAfter bug fixes or small changesTesting fixed functionality only
Alpha TestingTesting by internal team before releaseBefore beta testingInternal user testing
Beta TestingTesting by external users before releaseBefore final releaseReal user testing in production

Test Design Techniques

TechniqueDescriptionAdvantagesDisadvantagesBest For
Equivalence PartitioningDivide input data into partitions with same behaviorReduces test cases, efficientMay miss boundary errorsInput validation testing
Boundary Value AnalysisTest boundary values of input partitionsFinds boundary errors effectivelyLimited to numeric inputsRange-based inputs
Decision Table TestingTest combinations of inputs and actionsCovers all combinationsCan be complex for many inputsBusiness logic testing
State Transition TestingTest system behavior based on different statesGood for state-based systemsComplex for systems with many statesWorkflow applications
Error GuessingTest based on experience and intuitionFinds errors that other methods missDependent on tester experienceSupplement other techniques

Testing Approaches

ApproachDescriptionAdvantagesDisadvantagesBest For
Black Box TestingTesting without knowledge of internal codeFocuses on requirements, easy to designMay miss internal errorsFunctional testing
White Box TestingTesting with knowledge of internal code structureThorough code coverageRequires programming knowledgeUnit testing, security testing
Gray Box TestingCombination of black and white box testingBalanced approachRequires both skill setsIntegration testing
Manual TestingTesting performed by human testersExploratory, intuitive testingTime-consuming, prone to human errorExploratory, usability testing
Automated TestingTesting performed using tools/scriptsFast, repeatable, scalableInitial setup cost, maintenanceRegression, performance testing

Quality Assurance vs Quality Control

AspectQuality AssuranceQuality ControlKey Differences
FocusProcess improvementProduct verificationQA prevents defects, QC detects defects
TimingThroughout development lifecycleDuring and after developmentQA is proactive, QC is reactive
ActivitiesProcess definition, standardsTesting, inspections, reviewsQA defines how, QC checks what
ResponsibilityEntire organizationTesting/QA teamQA is everyone's responsibility
MeasurementProcess metricsProduct metricsQA measures processes, QC measures products

Test Automation Frameworks

Framework TypeDescriptionAdvantagesDisadvantagesBest For
Linear FrameworkSequential test execution without reusable componentsSimple to create and understandNot reusable, difficult to maintainSmall projects, quick automation
Modular FrameworkTests divided into independent modulesReusable, maintainableInitial design complexityMedium to large projects
Data-Driven FrameworkTest data separated from test scriptsSingle script tests multiple data setsRequires data managementTesting with multiple input values
Keyword-Driven FrameworkKeywords define test actionsNon-technical users can create testsComplex setup, maintenanceBusiness users creating tests
Hybrid FrameworkCombination of multiple framework typesFlexible, comprehensiveComplex, requires expertiseLarge, complex projects
