Data Warehousing Concepts

ConceptDescriptionCharacteristicsBenefits
OLTP vs OLAPOnline Transaction Processing vs Online Analytical ProcessingOLTP: Transactional, normalized, frequent updates; OLAP: Analytical, denormalized, infrequent updatesOLTP: Fast transactions; OLAP: Fast analysis
DimensionsDescriptive attributes of dataTime, product, location, customerContext for analysis
Fact TablesContain quantitative data and foreign keysMeasures, metrics, factsNumerical data for analysis
Star SchemaDimension tables connected to fact tableCentral fact table, surrounding dimension tablesSimple, fast queries
Snowflake SchemaNormalized version of star schemaNormalized dimension tablesReduces redundancy
Fact ConstellationMultiple fact tables sharing dimensionsShared dimensional tablesMultiple subject areas

Data Quality Issues

IssueDescriptionImpactSolution
Incomplete DataMissing values or attributesReduces analysis accuracyImputation, deletion, model missingness
Noisy DataData with errors or outliersDistorts analysis resultsSmoothing, binning, regression
Inconsistent DataSame information represented differentlyConfuses analysis algorithmsData standardization, normalization
Duplicate DataMultiple instances of same recordBiases analysis resultsDuplicate detection, merging
OutliersExtreme values far from othersSkews statistical measuresStatistical methods, clustering, visualization
