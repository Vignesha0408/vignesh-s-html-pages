Human-Computer Interaction (HCI) Cheat Sheet

HCI Core Principles

PrincipleDescriptionImplementationBenefits
VisibilitySystem status and options clearly visibleClear labels, status indicators, progress barsUsers know what's happening
FeedbackSystem provides feedback for user actionsConfirmation messages, loading indicators, sound effectsUsers know their actions were processed
ConstraintsLimit user options to prevent errorsDropdowns, checkboxes, disabled buttonsPrevents invalid actions
MappingRelationships between controls and effects are intuitiveLeft volume down, right volume upEasy to learn and use
ConsistencySimilar elements behave the same waySame icons, colors, layouts throughoutReduces learning curve
AccessibilityDesign for users with different abilitiesScreen readers, keyboard navigation, high contrastInclusive design for all users

Usability Heuristics (Nielsen)

HeuristicDescriptionExampleImplementation
Visibility of System StatusSystem keeps users informed about what's happeningLoading bar during file uploadProgress indicators, status messages
Match Between System and Real WorldUse familiar language and conceptsTrash can icon for deletionReal-world metaphors, common language
User Control and FreedomProvide undo/redo optionsBack button, cancel optionsEscape routes, undo functionality
Consistency and StandardsFollow platform conventionsStandard menu placementPlatform guidelines, consistent UI
Error PreventionDesign to prevent errorsConfirmation dialogs for deletionForm validation, clear warnings
Recognition Rather Than RecallMinimize user memory loadVisual cues, persistent navigationVisible options, breadcrumbs
Flexibility and Efficiency of UseAccommodate both novice and expert usersKeyboard shortcuts, customizable UIShortcuts, user preferences
Aesthetic and Minimalist DesignRemove irrelevant informationSimple, clean interfaceEssential information only
Help Users Recognize and Recover from ErrorsClear error messages"File not found" with suggestionClear, constructive error messages
Help and DocumentationProvide context-sensitive helpTooltips, help buttonsContextual help, documentation

User Research Methods

MethodPurposeWhen to UseAdvantagesDisadvantages
InterviewsUnderstand user needs and motivationsEarly design phase, requirements gatheringRich qualitative data, detailed insightsTime-consuming, potential bias
SurveysCollect quantitative data from many usersValidate hypotheses, measure satisfactionLarge sample size, statistical dataLimited depth, response bias
ObservationSee how users actually behaveContextual understanding, usability issuesReal behavior, no biasTime-consuming, observer effect
Focus GroupsExplore attitudes and opinionsConcept validation, feedback collectionGroup dynamics, brainstormingGroupthink, dominant voices
Usability TestingIdentify usability problemsDesign validation, iterationDirect observation of issuesArtificial setting, limited scenarios
Card SortingUnderstand mental modelsInformation architecture designReveals natural categorizationArtificial task, limited context

Interface Design Elements

ElementPurposeBest PracticesCommon Mistakes
NavigationHelp users move through the interfaceClear labels, consistent placement, breadcrumbsHidden navigation, inconsistent structure
FormsCollect user inputClear labels, logical grouping, validationUnclear labels, too many fields
ButtonsTrigger actionsClear labels, appropriate size, visual feedbackUnclear purpose, poor affordance
IconsVisual representation of functionsUniversal symbols, consistent style, tooltipsUnclear meaning, inconsistent style
TextCommunicate informationClear hierarchy, readable fonts, appropriate sizeToo much text, poor contrast
ColorProvide visual cues and hierarchyAccessible contrast, consistent meaningColor as only indicator, poor contrast

Evaluation Methods

MethodApproachWhen to UseAdvantagesDisadvantages
Heuristic EvaluationExperts review interface against heuristicsEarly design, quick assessmentFast, cost-effective, identifies issuesMay miss user-specific issues
Usability TestingReal users perform tasks with systemDesign validation, iterationReal user behavior, actual problemsExpensive, time-consuming
Think Aloud ProtocolUsers verbalize thoughts while using systemUnderstanding user thought processInsight into user thinkingMay alter natural behavior
Cognitive WalkthroughStep-by-step analysis of user tasksEarly design, task-focused evaluationFocus on learnabilityTime-consuming, limited scope
A/B TestingCompare two versions of interfaceOptimizing existing designsQuantitative comparison, real dataRequires large user base, ethical concerns
