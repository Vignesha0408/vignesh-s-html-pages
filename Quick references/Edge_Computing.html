Edge Computing - Quick Reference

ConceptDescription
Definition
Computing paradigm that brings computation and data storage closer to the location where it is needed, typically at the edge of the network near data sources.
Purpose
Reduce latency, bandwidth usage, and improve response times by processing data closer to its source.
Key Benefits
Low latency, reduced bandwidth, improved privacy, enhanced reliability
Architecture ComponentsDescription
Edge Devices
Sensors, IoT devices, smartphones, routers that collect and process data locally.
Edge Gateways
Intermediate devices that aggregate data from edge devices and perform initial processing.
Edge Servers
Local computing infrastructure that runs applications and processes data near the source.
Cloud Core
Central cloud infrastructure for storage, analytics, and management of edge resources.
Edge vs Cloud ComputingDescription
Edge Computing
Processing occurs near data source, low latency, limited compute resources
Cloud Computing
Processing occurs in centralized data centers, high latency, massive compute resources
Hybrid Approach
Combines both for optimal performance and resource utilization
Key TechnologiesDescription
Containerization
Docker, Kubernetes for deploying applications at the edge with consistency.
5G Networks
Ultra-low latency networks supporting edge computing applications.
Virtualization
Network Function Virtualization (NFV) and Software Defined Networking (SDN).
AI/ML at Edge
Machine learning models deployed and executed on edge devices.
ApplicationsDescription
Autonomous Vehicles
Real-time processing for navigation, obstacle detection, and decision making.
Smart Cities
Traffic management, surveillance, environmental monitoring.
Industrial IoT
Predictive maintenance, quality control, process optimization.
Healthcare
Remote patient monitoring, real-time diagnostics, telemedicine.
ChallengesDescription
Resource Constraints
Limited processing power, memory, and storage on edge devices.
Security
Distributed nature creates more attack vectors and security challenges.
Management
Managing large number of distributed edge devices and applications.
Data Consistency
Maintaining consistency across distributed edge nodes.
Edge Computing PlatformsDescription
AWS IoT Greengrass
Extends AWS cloud capabilities to edge devices for local processing.
Azure IoT Edge
Microsoft's platform for running cloud workloads on edge devices.
Google Cloud IoT Edge
Google's edge computing platform with AI/ML capabilities.
KubeEdge
Open-source system for extending Kubernetes to edge nodes.
Performance MetricsDescription
Latency
Time delay between data generation and processing response.
Bandwidth
Amount of data transferred between edge and cloud.
Reliability
Consistency and uptime of edge computing services.
Scalability
Ability to handle increasing number of edge devices and workloads.
