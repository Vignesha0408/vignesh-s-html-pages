Sensor Networks - Quick Reference

ConceptDescription
Definition
Network of spatially distributed autonomous sensors to monitor physical or environmental conditions and cooperatively pass their data through the network.
Components
Sensor nodes, sink nodes, base stations, communication protocols
Characteristics
Resource constraints, self-organization, multi-hop communication, scalability
Sensor Node ArchitectureDescription
Sensing Unit
Collects data from environment using sensors (temperature, humidity, light, etc.)
Processing Unit
Microcontroller for data processing and decision making
Communication Unit
Radio transceiver for wireless communication
Power Unit
Battery or energy harvesting for power supply
Types of SensorsDescription
Physical Sensors
Temperature, pressure, humidity, light, sound sensors
Chemical Sensors
Gas, pH, chemical composition sensors
Biological Sensors
Heart rate, blood pressure, glucose level sensors
Motion Sensors
Accelerometers, gyroscopes, magnetometers
Network TopologiesDescription
Star Topology
All nodes connect directly to a central node (sink/base station)
Mesh Topology
Multi-hop network where nodes relay data for other nodes
Tree Topology
Hierarchical structure with parent-child relationships
Cluster-Based
Network divided into clusters with cluster heads
Communication ProtocolsDescription
MAC Protocols
TDMA, CSMA, S-MAC, T-MAC for medium access control
Routing Protocols
LEACH, PEGASIS, TEEN, SPIN for energy-efficient routing
Transport Protocols
Specialized protocols for reliable data delivery
ApplicationsDescription
Environmental Monitoring
Weather stations, pollution monitoring, forest fire detection
Healthcare
Patient monitoring, body area networks, telemedicine
Smart Cities
Traffic management, waste management, smart parking
Agriculture
Soil monitoring, irrigation control, crop monitoring
ChallengesDescription
Energy Efficiency
Power constraints require energy-efficient protocols and algorithms
Scalability
Networks must handle thousands of nodes efficiently
Reliability
Handling node failures and environmental conditions
Security
Protecting against attacks in resource-constrained environment
Energy ManagementDescription
Power Conservation
Duty cycling, sleep/wake scheduling, power-aware routing
Energy Harvesting
Solar, thermal, kinetic energy collection
Data Aggregation
Reducing transmission by combining data at intermediate nodes
