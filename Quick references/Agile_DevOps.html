Agile & DevOps - Quick Reference

ConceptDescription
Agile
Software development methodology emphasizing iterative development, collaboration, and customer feedback.
DevOps
Culture and practice that unifies software development and operations teams for faster delivery.
Agile Manifesto
Individuals & interactions, Working software, Customer collaboration, Responding to change
Agile FrameworksDescription
Scrum
Framework with roles (Product Owner, Scrum Master, Team), events (Sprint, Daily Scrum, Review, Retrospective), and artifacts (Product Backlog, Sprint Backlog, Increment).
Kanban
Visual workflow management method using boards and cards to visualize work and limit WIP.
Extreme Programming (XP)
Focuses on engineering practices like pair programming, TDD, continuous integration.
Lean
Eliminates waste, amplifies learning, decides as late as possible, delivers as fast as possible.
Scrum RolesDescription
Product Owner
Represents stakeholders, manages Product Backlog, defines requirements, prioritizes features.
Scrum Master
Facilitates Scrum process, removes impediments, coaches team, ensures proper implementation.
Development Team
Cross-functional team that develops the product, self-organizing, collectively responsible.
DevOps PracticesDescription
CI/CD
Continuous Integration (frequent code merging) and Continuous Delivery (automated deployment).
Infrastructure as Code (IaC)
Managing infrastructure through machine-readable files instead of manual processes.
Monitoring & Logging
Real-time visibility into application and infrastructure performance.
Automated Testing
Unit, integration, and end-to-end tests integrated into deployment pipeline.
DevOps ToolsDescription
Version Control
Git, SVN for source code management and collaboration.
CI/CD
Jenkins, GitLab CI, CircleCI, GitHub Actions for automation pipelines.
Containerization
Docker for packaging applications, Kubernetes for container orchestration.
Cloud Platforms
AWS, Azure, GCP for infrastructure and deployment services.
Agile CeremoniesDescription
Sprint Planning
Team plans work for upcoming sprint, defines Sprint Goal.
Daily Standup
15-minute meeting to synchronize team, discuss progress and blockers.
Sprint Review
Demonstrate completed work to stakeholders, gather feedback.
Sprint Retrospective
Team reflects on sprint, identifies improvements for next sprint.
DevOps BenefitsDescription
Faster Delivery
Shorter development cycles, faster time to market.
Improved Quality
Automated testing reduces defects, better monitoring detects issues early.
Enhanced Collaboration
Breaks down silos between development and operations teams.
Increased Reliability
Infrastructure as code ensures consistency and reduces human error.
Key MetricsDescription
Lead Time
Time from work start to delivery to customer.
Deployment Frequency
How often deployments occur to production.
Mean Time to Recovery
Time to restore service after failure.
Change Failure Rate
Percentage of deployments causing failures.
