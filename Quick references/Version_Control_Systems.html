Version Control Systems - Quick Reference

ConceptDescription
Definition
System that records changes to files over time so you can recall specific versions later. Essential for collaboration and code management.
Purpose
Track changes, collaborate, maintain history, enable branching/merging, backup and restore
Types
Distributed (Git) vs Centralized (SVN), Client-server vs Peer-to-peer
Git BasicsDescription
Initialize
git init (create repo), git clone (copy remote repo)
Add & Commit
git add (stage changes), git commit -m "message" (save changes)
Status & Log
git status (check status), git log (view commit history)
Push & Pull
git push (upload to remote), git pull (download from remote)
Git BranchingDescription
Create & Switch
git branch (list branches), git checkout -b new-branch (create and switch), git switch new-branch (switch)
Merge & Rebase
git merge (combine branches), git rebase (reapply commits on top of another base)
Remote Branches
git push origin branch-name (push branch), git fetch (get remote changes)
Git AdvancedDescription
Stash
git stash (save changes temporarily), git stash pop (apply stashed changes)
Reset & Revert
git reset (undo changes), git revert (create new commit to undo changes)
Tagging
git tag (list tags), git tag -a v1.0 -m "message" (create annotated tag)
Log & Diff
git diff (show differences), git log --oneline (short log format)
SVN BasicsDescription
Checkout
svn checkout (get working copy), svn update (get latest changes)
Add & Commit
svn add (add files), svn commit -m "message" (save changes)
Status & Log
svn status (check status), svn log (view commit history)
Update
svn update (get latest changes from repository)
SVN Branching & MergingDescription
Branching
svn copy trunk branches/new-branch -m "Create branch" (create branch)
Switching
svn switch (switch working copy to different branch)
Merging
svn merge (combine changes from branches)
Git vs SVN ComparisonDescription
Architecture
Git: Distributed, each clone is full repository; SVN: Centralized, single server with working copies
Performance
Git: Faster operations since most work done locally; SVN: Slower, requires network for most operations
Branching
Git: Lightweight, fast branching; SVN: More expensive, stored on server
History
Git: Immutable SHA-1 hashes; SVN: Incremental revision numbers
Best PracticesDescription
Commit Messages
Use clear, descriptive messages, follow conventions (e.g., conventional commits)
Branch Strategy
Feature branching, Git Flow, Trunk-based development
Workflow
Regular commits, pull before push, use branches for features, review code before merge
Backup
Push regularly to remote repositories, use multiple remotes
