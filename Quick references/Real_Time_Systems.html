Real-Time Systems - Quick Reference

ConceptDescription
Definition
Systems that must respond to inputs within strict time constraints. Correctness depends on both logical and temporal correctness.
Characteristics
Time constraints, predictability, reliability, resource constraints
Applications
Aerospace, automotive, medical devices, industrial control, multimedia
System TypesDescription
Hard Real-Time
Missing deadlines has catastrophic consequences. Strict timing requirements.
Soft Real-Time
Missing deadlines degrades performance but doesn't cause system failure.
Firm Real-Time
Results are useless after deadline, but no catastrophic consequences.
Task SchedulingDescription
Rate Monotonic Scheduling (RMS)
Static priority scheduling based on task periods. Shorter period = higher priority.
Earliest Deadline First (EDF)
Dynamic priority scheduling. Task with earliest deadline has highest priority.
Deadline Monotonic Scheduling
Similar to RMS but based on relative deadlines rather than periods.
Fixed Priority Scheduling
Static priorities assigned at design time.
Timing ConstraintsDescription
Deadline
Time by which task must complete. Hard or soft constraints.
Period
Time interval between consecutive task releases.
Execution Time
Time required to complete a task. WCET (Worst Case Execution Time).
Release Time
Time when task becomes available for execution.
Operating SystemsDescription
RTOS
Real-Time Operating Systems (VxWorks, QNX, FreeRTOS, RTLinux)
Preemptive Kernel
Kernel that can interrupt running tasks to execute higher priority tasks.
Deterministic Services
Predictable timing for OS services like task switching, interrupt handling.
Resource ManagementDescription
Priority Inversion
Low-priority task blocks high-priority task by holding shared resource.
Priority Inheritance
Low-priority task inherits priority of blocked high-priority task.
Priority Ceiling Protocol
Resources have priority ceilings to prevent priority inversion.
Design ConsiderationsDescription
Predictability
System behavior must be analyzable and predictable.
Determinism
Operations must have bounded execution times.
Minimal Jitter
Consistent timing with minimal variation.
Resource Isolation
Prevent interference between critical and non-critical tasks.
Analysis TechniquesDescription
Rate Monotonic Analysis
Mathematical analysis to determine if task set is schedulable.
Deadline Monotonic Analysis
Analysis for tasks with deadlines different from periods.
Response Time Analysis
Calculate worst-case response time for each task.
Schedulability Tests
Tests to determine if all tasks meet their deadlines.
